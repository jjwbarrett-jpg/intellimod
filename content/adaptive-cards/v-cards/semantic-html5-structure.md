### Summary
Semantic HTML5 tags improve document structure, accessibility, and SEO. Using elements like main, article, and section conveys intent to browsers and assistive technology. Structure should follow a logical outline using headings and nested semantic elements.

### Core Principles
   - Tags should reflect meaning, not style.
   - header, nav, main, footer are structural anchors.
   - article, section, aside provide content segmentation.
   - figure and figcaption add multimedia context.

### Best Practices
   - Use semantic tags wherever applicable.
   - Avoid overusing div and span for structure.
   - Follow heading hierarchy (do not use h3 before h2).

### Use Cases
   - Blog posts.
   - Landing pages.
   - Dashboards.
   - Screen reader–friendly UIs.
